# NPI Tracker - Django + Vite Integration

A Django project configured with django-vite for modern asset bundling and hot module reloading (HMR).

## Features

- **Django 4.2+** - Modern Python web framework
- **Vite 5** - Lightning-fast frontend build tool
- **django-vite** - Seamless integration between Django and Vite
- **Hot Module Reloading (HMR)** - Instant feedback during development
- **Whitenoise** - Efficient static file serving in production

## Setup Instructions

### 1. Install Python Dependencies

```bash
# Create a virtual environment (recommended)
python -m venv venv

# Activate virtual environment
# On Windows:
venv\Scripts\activate
# On macOS/Linux:
# source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### 2. Install Node.js Dependencies

```bash
npm install
```

### 3. Run Database Migrations

```bash
python manage.py migrate
```

### 4. Create a Superuser (Optional)

```bash
python manage.py createsuperuser
```

## Development Mode (with Hot Reload)

Run both Django and Vite development servers simultaneously:

### Terminal 1 - Start Vite Dev Server

```bash
npm run dev
```

This will start the Vite development server on `http://localhost:5173` with HMR enabled.

### Terminal 2 - Start Django Server

```bash
python manage.py runserver
```

This will start the Django development server on `http://127.0.0.1:8000`.

Now open `http://127.0.0.1:8000` in your browser. Any changes you make to the frontend files (`frontend/src/`) will be instantly reflected without page reload thanks to HMR!

## Production Build

### 1. Build Frontend Assets

```bash
npm run build
```

This compiles your frontend assets and creates a `manifest.json` file in the `assets/` directory.

### 2. Collect Static Files

```bash
python manage.py collectstatic --noinput
```

### 3. Set Production Settings

In `config/settings.py`, set:
```python
DEBUG = False
ALLOWED_HOSTS = ['yourdomain.com']
```

### 4. Run with Production Server

```bash
# Example with gunicorn
pip install gunicorn
gunicorn config.wsgi:application
```

## How django-vite Works

### In Development Mode (`DEBUG = True`)

- **Vite dev server** runs on port 5173
- **Django templates** load assets directly from Vite dev server
- **HMR is enabled** - changes are reflected instantly
- The `{% vite_hmr_client %}` tag injects the Vite HMR client

### In Production Mode (`DEBUG = False`)

- Assets are **pre-built** with `npm run build`
- Django serves **static files** from the `assets/` directory
- The `manifest.json` maps source files to their hashed versions
- No Vite dev server needed

## Template Usage

The base template (`templates/base.html`) demonstrates django-vite template tags:

```django
{% load django_vite %}
<!DOCTYPE html>
<html>
<head>
    {% vite_hmr_client %}
    {% vite_asset 'frontend/src/main.js' %}
</head>
<body>
    <div id="app">
        {% block content %}{% endblock %}
    </div>
</body>
</html>
```

- `{% vite_hmr_client %}` - Injects HMR client in development mode
- `{% vite_asset 'path/to/file.js' %}` - Loads JS/CSS assets appropriately for dev/prod

## Configuration

### Django Settings (`config/settings.py`)

```python
DJANGO_VITE = {
    'default': {
        'dev_mode': DEBUG,
        'dev_server_host': 'localhost',
        'dev_server_port': 5173,
        'static_url_prefix': 'static',
        'manifest_path': BASE_DIR / 'assets' / 'manifest.json',
    }
}
```

### Vite Config (`vite.config.js`)

```javascript
export default defineConfig({
  base: '/static/',
  build: {
    manifest: 'manifest.json',
    outDir: resolve('./assets'),
    rollupOptions: {
      input: {
        main: resolve('./frontend/src/main.js'),
      },
    },
  },
});
```

## Adding New Assets

1. Create your JS/CSS file in `frontend/src/`
2. Add it to `vite.config.js` input:
```javascript
input: {
  main: resolve('./frontend/src/main.js'),
  admin: resolve('./frontend/src/admin.js'),  // New entry
}
```
3. Load it in your template:
```django
{% vite_asset 'frontend/src/admin.js' %}
```

## Troubleshooting

### Assets not loading in development

- Ensure Vite dev server is running (`npm run dev`)
- Check that port 5173 is accessible
- Verify `DJANGO_VITE['default']['dev_mode']` is `True`

### Assets not loading in production

- Run `npm run build` to compile assets
- Ensure `manifest.json` exists in `assets/` directory
- Run `python manage.py collectstatic`

### HMR not working

- Check browser console for WebSocket connection errors
- Ensure Vite dev server is running
- Verify `{% vite_hmr_client %}` is in your template
