How to connect Power Automate

Works with free Power Automate — uses the OneDrive for Business "Upload file from URL" action. No Premium, no admin needed.

  1. Create an inbound webhook below — choose the action and copy the Endpoint URL
  2. Create a folder in OneDrive called /PA_Webhooks
  3. In Power Automate, add the "Upload file from URL" action (OneDrive for Business — free connector)
  4. Set Source URL to the endpoint with your data as query parameters: https://your-domain/api/inbound/<token>/?title=My+Issue&severity=critical&project_name=Project+Alpha Use dynamic content from your trigger to fill in the values
  5. Set Destination File Path: /PA_Webhooks/result.json
  6. Set Overwrite: Yes
  7. Run the flow — OneDrive fetches your URL, Django processes the event, the response is saved as a file

How it works: The "Upload file from URL" action tells OneDrive to download content from the Source URL. OneDrive makes a GET request to your endpoint — Django processes the webhook and returns a JSON response. The response gets saved as a file in OneDrive (you can ignore or delete it).

Free — email triggers the flow, AI (Google Gemini) parses the email body into a structured issue. No strict format required.

  1. Create a Create Issue inbound webhook below and copy the Endpoint URL
  2. In Power Automate, add trigger: "When a new email arrives (V3)" (Outlook)
  3. Set Subject Filter to NPI: to only process relevant emails
  4. Add a Compose action named LatestReply with expression: first(split(triggerOutputs()?['body/bodyPreview'], 'From:')) This extracts only the latest reply, stripping the thread history
  5. Add "Upload file from URL" (OneDrive for Business) with Source URL expression: concat('https://your-domain/api/inbound/<token>/?raw_text=', encodeUriComponent(outputs('LatestReply')), '&project_name=', encodeUriComponent('Your Project'))
  6. Set Destination File Path: /PA_Webhooks/result.json, Overwrite: Yes
  7. Django receives the email text → Gemini AI extracts title, severity, impact, etc. → issue is created automatically

How it works: When raw_text is passed without a title, the server uses Google Gemini to parse the free-text into structured issue fields. Users can write emails in plain language — no special format needed. Requires GOOGLE_API_KEY env variable.

Requires Power Automate Premium — the built-in HTTP action is a premium connector.

  1. Create an inbound webhook below — choose the action Power Automate should trigger
  2. Copy the Endpoint URL shown on the webhook card
  3. In Power Automate, add an HTTP action (POST) pointing to https://your-domain/api/inbound/<token>/
  4. Set Content-Type header to application/json
  5. Set the body to JSON matching the expected payload for the chosen action
  6. Run the flow — NPI Tracker processes the event instantly
