Phase-Gate + Kanban Hybrid
Phase-Gate (Sequential Stages)
NPI inherently follows sequential build stages EVT → DVT → PVT → MP. Each stage has gate readiness criteria that must be met before proceeding. This maps naturally to the Phase-Gate model (modified Waterfall). Your existing Build Stages with gate checklists already implement this.
Kanban (Within Each Phase)
Within each phase, day-to-day task execution benefits from Kanban flow — tasks move through statuses (Open → In Progress → Done/Blocked). The Critical Index acts as a prioritized backlog, focusing the team on the highest-impact work.
Why Not Pure Agile/Scrum?
NPI has hard external dependencies — tooling lead times (4-8 weeks), supplier schedules, customer approval gates — that don't map to fixed-length sprints. The work is driven by critical path constraints, not sprint velocity.
